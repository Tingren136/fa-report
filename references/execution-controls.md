# 阶段门槛、压力测试与质量控制

## 1. 阶段门槛

| 阶段 | 必须满足 | 允许输出 |
|---|---|---|
| 预调查 | 主体可唯一识别；已盘点资料；已披露检索和材料边界 | 是否继续、P0补件、初步风险，不定额度 |
| 授信分析 | 全部经营账户覆盖；十二个月本息可测；融资金额、用途和期限明确 | 初步额度/期限边界、正式 DSCR、风控条件 |
| 完整授信方案 | 最新征信、关联风险、用途凭证、第二还款来源权属/估值/顺位核清 | 完整产品、额度、期限、担保及放款条件 |

出现以下事项时，不得自动上调报告阶段：主体冲突未解除、未结清逾期、重大造假疑点、全部经营账户不可获得、融资用途无法说明。门槛用于控制结论强度，不代替资金方制度。

## 2. 调查覆盖评分

按模块分别给出，不把总分当作审批结论：

- 完整度：必要材料已取得比例。
- 权威性：底层登记/原始凭证/权威来源占比。
- 时效性：材料是否覆盖调查基准日。
- 一致性：不同来源是否完成勾稽且无未解释冲突。

每项使用 `完整/部分/缺失`，并列出导致降级的具体证据，不使用伪精确综合分数。

## 3. 压力测试

达到授信分析阶段后至少测试：毛利率下降、回款延迟、主要客户流失、续贷失败、利率上升。参数必须可追溯到资金方政策、历史波动或明确假设。分别展示基准、温和压力和严重压力下的可用于偿债现金流、债务本息和 DSCR。

## 4. 发布前自动检查

运行 `scripts/validate_case.py` 检查案件阶段，运行 `scripts/validate_report.py` 检查 Markdown 报告主题、证据编号和敏感信息。人工复核以下事项：

1. 首页结论与正文风险、红线和阶段一致。
2. 关键数字具有证据编号和精确定位。
3. 财报、税表、开票和流水差异已量化并解释。
4. 图中的确定关系均有证据；线索使用虚线。
5. 公式、加总、单位、币种和期间一致。
6. 补件具有责任人、截止日期、验收标准和重算模块。
7. 未把搜索摘要、企业自述或“未检索到”写成已核实事实。
8. 正式报告已脱敏。
