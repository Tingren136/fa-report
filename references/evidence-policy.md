# 证据与联网检索规则

## 来源优先级

1. 政府、监管机构、法院、交易所、法定登记或公示系统。
2. 企业正式公告、审计报告、募集说明书及有签章的原始业务资料。
3. 金融机构、行业协会、权威研究机构的正式资料。
4. 主流媒体和专业数据库。
5. 企业官网、公众号、自媒体、聚合搜索摘要和论坛。

低优先级来源可用于发现线索，不应单独支撑重大负面结论。涉及诉讼、处罚、失信、股权或财务数字时，尽量回溯一手来源。

## 证据台账字段

| 字段 | 说明 |
|---|---|
| 编号 | 本地 `L-001`；网络 `W-001` |
| 主体 | 该证据对应的法人、自然人或资产 |
| 主题 | 工商、司法、财务、流水、征信、担保等 |
| 来源 | 文件名或网页标题 |
| 定位 | 页码、表名、单元格、交易行或网页章节 |
| 日期 | 文件期间、发布日期或查询日期 |
| 摘要 | 只写证据直接证明的事实 |
| 状态 | 已核实、企业提供、计算结果、分析推断、信息矛盾、待补资料 |
| 链接 | 网络来源使用直接页面链接 |

## 搜索策略

- 先按 `source-routing.md` 将股权、司法、处罚、知识产权、招投标和行业监管分别路由到专业网站；通用搜索引擎不计作专业平台已覆盖。
- 使用企业完整名称、统一社会信用代码、历史名称分别检索。
- 将实控人、法定代表人、主要股东和关键关联企业纳入检索。
- 对每个主体组合使用：处罚、诉讼、执行、失信、限高、欠税、环保、事故、股权质押、动产抵押、破产、注销、舆情等关键词。
- 对行业环境检索最新政策、价格、供需、周期和重大监管变化，并注明数据期间。
- 当前身份、状态、金额、法规、政策或市场数据属于易变信息，必须联网核验并写明查询日期。
- 第三方企业图谱中“疑似实控人、最终受益人、关联企业、风险数量”等算法结果属于线索；只有取得底层登记、案件或监管记录后，才能升级为已核实事实。

## 冲突处理

1. 检查是否为同一主体、同一期间、同一币种和同一统计口径。
2. 原样记录冲突双方，不擅自选择对企业更有利的一方。
3. 优先采用更权威、更接近原始记录、更新日期更近的来源。
4. 能计算时列出差额和差异率；不能解释时标记“信息矛盾”。
5. 将解除冲突所需的材料列入补件清单。

## 引用和表述

- 外部事实：正文近处标注来源标题、发布日期或查询日期，并附直接链接。
- 本地事实：标注证据编号和精确定位，不在公开报告中暴露完整账号或个人证件号码。
- 推断表述使用“可能、提示、需关注、尚待核实”；避免把推断写成确定事实。
- “未检索到”只代表本次覆盖范围内未发现，不得改写为“没有”。
- 搜索结果摘要、缓存片段或第三方转载若无法打开原文，须降低证据等级并披露限制。
